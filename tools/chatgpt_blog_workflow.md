# ChatGPT 블로그 작업 운영 기준

이 문서는 `kerogrammer/kerogrammer.github.io` 저장소에서 ChatGPT가 블로그 작업을 수행할 때 따라야 할 **고정 운영 기준**을 정리한 문서다.
세션이 바뀌더라도, 이후 작업은 이 문서를 먼저 확인한 뒤 진행한다.

---

## 1. 기본 원칙

- 블로그 글 작업은 **채팅에서 초안 확인 → 사용자 승인 1회 → main에 한 번에 반영** 방식으로 진행한다.
- 사용자가 별도로 요청하지 않는 한, 불필요한 중간 쓰기 작업을 만들지 않는다.
- 사용자가 승인 버튼을 여러 번 누르게 만드는 흐름은 피한다.

---

## 2. 기본 작업 절차

### 표준 절차
1. ChatGPT가 채팅창에 포스팅 초안을 먼저 제시한다.
2. 사용자가 내용 확인 후 수정 요청 또는 승인 의사를 밝힌다.
3. 승인되면 다음 항목을 **가능한 한 한 번의 반영으로** 처리한다.
   - 포스트 파일 생성 또는 수정
   - 썸네일 파일 추가 또는 수정
   - front matter 정리
4. 반영 후에는 결과만 간단히 보고한다.

### 하지 말아야 할 것
- 사용자가 요청하지 않았는데 브랜치 생성
- 사용자가 요청하지 않았는데 Draft PR 생성
- 사용자가 요청하지 않았는데 PR 코멘트 작성
- 정리용 메모를 남긴다는 이유로 추가 쓰기 작업 수행

---

## 3. 포스팅 반영 방식

- 기본 반영 대상 브랜치는 `main`이다.
- 사용자가 따로 검토용 브랜치나 PR을 원할 때만 예외적으로 사용한다.
- 썸네일이 필요한 경우, 가능하면 포스트 반영과 **같은 작업 흐름 안에서 함께 반영**한다.
- 미래 시각 포스팅으로 배포 누락이 생기지 않도록, 발행 시점 기준으로 안전한 `date` 값을 사용한다.

---

## 4. 이 저장소의 블로그 구조 메모

- 블로그 엔진: **Jekyll + Chirpy**
- 포스트 위치: `_posts/`
- 초안 위치: `_drafts/`
- 썸네일 예시 경로: `assets/img/posts/`
- timezone: `Asia/Seoul`

---

## 5. 사용자 선호 요약

- 불필요한 승인 요청을 싫어한다.
- 초안은 먼저 채팅에서 보고 싶어 한다.
- 승인 후에는 한 번에 깔끔하게 반영되길 원한다.
- 부수적인 코멘트, PR, 정리 작업은 사용자가 요청할 때만 수행한다.

---

## 6. 다음 세션에서의 사용 방법

새 세션에서 사용자가 블로그 작업을 요청하면, ChatGPT는 먼저 이 문서를 참고해 아래처럼 동작한다.

- "이 저장소의 운영 기준에 맞춰 초안을 먼저 보여드리겠습니다. 승인 주시면 main에 한 번에 반영하겠습니다."

즉, 세션이 달라져도 이 문서를 기준으로 동일한 작업 방식을 유지한다.
