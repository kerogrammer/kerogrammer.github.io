# SEO and Accessibility Best Practices

Standardizing SEO and accessibility to improve search engine ranking and user experience.

## Metadata (YAML)
- **Description**: Every post must have a `description` field in the front matter. It should be 120-160 characters long and include primary keywords.
- **Title**: Ensure the `title` is descriptive and reflects the content accurately.
- **Tags**: Use relevant tags to assist with internal search and discovery.

## Content Structure
- **Heading Order**: Always use headings in sequential order (`##` before `###`). Never skip levels.
- **Single H1**: The title defined in front matter is rendered as `<h1>`. Do not use `<h1>` within the markdown body.
- **Semantic HTML**: Use standard markdown elements to ensure Jekyll renders semantic HTML5 tags.

## Images
- **Alt Text**: All images must have an `alt` attribute that describes the image for screen readers and search engines.
- **Image Size**: Optimize image sizes (WebP) to ensure fast page load times, which is a key SEO ranking factor.

## Linking
- **Internal Links**: Use internal links to other posts to build a cohesive site structure.
- **External Links**: Ensure external links are high quality and relevant.
- **Self-Descriptive Links**: Avoid "click here". Instead, use "read about the [Dune Imperium Season 28 Game Results](...)".

## Verification
- Use titles that are likely to be searched (e.g., "Dune Imperium Digital League Review" instead of just "Game 3").
